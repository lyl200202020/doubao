## 四、关键技术实现

### 4.1 密码加密

使用 SM3 国密算法：
```java
String encryptedPassword = SM3Util.hash(password);
```

### 4.2 逻辑删除

所有实体添加 `@TableLogic` 注解，查询自动过滤 deleted=0 的记录。

### 4.3 树形结构

递归构建树形结构（机构树、菜单树）：
```java
private List<MenuVO> buildMenuTree(List<SysMenu> allMenus, String parentId) {
    return allMenus.stream()
        .filter(menu -> parentId == null ? menu.getParentMenuId() == null : parentId.equals(menu.getParentMenuId()))
        .map(menu -> {
            MenuVO vo = convertToVO(menu);
            vo.setChildren(buildMenuTree(allMenus, menu.getMenuId()));
            return vo;
        })
        .collect(Collectors.toList());
}
```

### 4.4 事务管理

所有写操作添加 `@Transactional(rollbackFor = Exception.class)` 注解。

### 4.5 参数校验

使用 Jakarta Validation 注解：
```java
@NotBlank(message = "Role code is required")
private String roleCode;
```

## 五、数据库表结构

### 5.1 表清单

| 表名 | 说明 | 主键 |
|------|------|------|
| sys_org | 机构表 | org_id |
| sys_user | 用户表 | user_id |
| sys_role | 角色表 | role_id |
| sys_menu | 菜单表 | menu_id |
| sys_user_role | 用户 - 角色关联表 | id (自增) |
| sys_menu_role | 菜单 - 角色关联表 | id (自增) |

### 5.2 关键字段

**sys_org**:
- org_id: 机构 ID（UUID）
- org_code: 机构代码（唯一）
- org_name: 机构名称
- org_level: 机构级别（0-总行，1-分行，2-支行，3-分理处）
- parent_org_id: 上级机构 ID
- org_status: 状态（0-停用，1-启用）
- deleted: 逻辑删除标记

**sys_user**:
- user_id: 用户 ID（UUID）
- user_code: 用户工号（唯一）
- user_name: 用户姓名
- password: 密码（SM3 加密）
- org_id: 所属机构 ID
- role_id: 所属角色 ID
- auth_level: 授权级别（1-16 级）
- user_status: 状态（0-停用，1-启用）

**sys_menu**:
- menu_id: 菜单 ID（UUID）
- menu_name: 菜单名称
- menu_type: 类型（1-目录，2-菜单，3-按钮）
- parent_menu_id: 父菜单 ID
- menu_url: 菜单 URL
- perms: 权限标识
- visible: 是否可见（0-隐藏，1-显示）
