## 二、核心表结构

### 2.1 sys_user（用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | varchar(32) | 用户ID（主键） |
| user_code | varchar(50) | 用户编码（登录账号） |
| real_name | varchar(100) | 真实姓名 |
| password | varchar(256) | 密码（SM3加密） |
| org_id | varchar(32) | 所属机构ID |
| auth_level | tinyint | 授权层级（1-10） |
| user_status | tinyint | 状态（1启用/0禁用） |
| last_login_time | datetime | 最后登录时间 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |
| deleted | tinyint | 删除标记（0正常/1删除） |

### 2.2 sys_role（角色表）

| 字段 | 类型 | 说明 |
|------|------|------|
| role_id | varchar(32) | 角色ID（主键） |
| role_code | varchar(50) | 角色编码（唯一） |
| role_name | varchar(100) | 角色名称 |
| role_status | tinyint | 状态（1启用/0禁用） |
| role_desc | varchar(256) | 角色描述 |
| created_by | varchar(32) | 创建人 |
| created_time | datetime | 创建时间 |
| updated_by | varchar(32) | 更新人 |
| updated_time | datetime | 更新时间 |
| deleted | tinyint | 删除标记（0正常/1删除） |

### 2.3 sys_menu（菜单表）

| 字段 | 类型 | 说明 |
|------|------|------|
| menu_id | varchar(32) | 菜单ID（主键） |
| parent_menu_id | varchar(32) | 父菜单ID（树结构，根节点为'0'） |
| menu_name | varchar(100) | 菜单名称 |
| **menu_type** | tinyint | **类型：1目录/2菜单/3按钮** |
| menu_url | varchar(256) | 路由路径 |
| perms | varchar(128) | 权限标识（如：sys:user:add） |
| menu_icon | varchar(50) | 图标名称 |
| sort_order | int | 排序号 |
| visible | tinyint | 是否可见（1可见/0隐藏） |
| created_by | varchar(32) | 创建人 |
| created_time | datetime | 创建时间 |
| updated_by | varchar(32) | 更新人 |
| updated_time | datetime | 更新时间 |
| deleted | tinyint | 删除标记（0正常/1删除） |

### 2.4 sys_user_role（用户-角色关联表）

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | varchar(32) | 用户ID（联合主键） |
| role_id | varchar(32) | 角色ID（联合主键） |

### 2.5 sys_role_menu（角色-菜单关联表）

| 字段 | 类型 | 说明 |
|------|------|------|
| role_id | varchar(32) | 角色ID（联合主键） |
| menu_id | varchar(32) | 菜单ID（联合主键） |

## 三、核心概念：菜单类型（menu_type）

| 类型值 | 名称 | 说明 | 示例 |
|--------|------|------|------|
| 1 | 目录 | 一级菜单，有子菜单，有图标，不可点击跳转 | 系统管理、客户管理 |
| 2 | 菜单 | 二级菜单，可点击跳转，对应具体页面 | 用户管理、角色管理 |
| 3 | 按钮 | 操作权限，菜单树中不可见，用于按钮级别权限控制 | 新增用户、删除用户 |

## 六、权限校验流程

### 6.1 登录流程

```
用户登录
    │
    ▼
验证用户名密码（SM3加密比对）
    │
    ▼
查询 sys_user_role 获取用户角色列表
    │
    ▼
查询 sys_role_menu 获取角色关联的菜单ID列表
    │
    ▼
查询 sys_menu 构建菜单树
    │
    ├── menu_type = 1,2 → 返回给前端渲染侧边栏
    │
    └── menu_type = 3 → 权限标识存入 permissions 列表
                        │
                        ▼
                前端 v-permission 指令控制按钮显示
```

### 6.2 前端权限控制

```typescript
// 路由守卫校验
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  if (!token) {
    next('/login')
  } else {
    next()
  }
})

// 按钮权限指令
<el-button v-permission="['sys:user:add']">新增用户</el-button>
```

### 6.3 后端权限校验

```java
@PreAuthorize("hasAuthority('sys:user:add')")
@PostMapping("/user")
public Result<Void> addUser(@RequestBody UserDTO userDTO) {
    // 业务逻辑
}
```

## 八、注意事项

1. **菜单树构建**：根节点的 `parent_menu_id` 为 `'0'`，不是 NULL
2. **权限继承**：角色必须同时拥有一级菜单和子菜单的权限才能正常显示
3. **按钮权限**：`menu_type=3` 的按钮权限通过 `perms` 字段标识，格式为 `模块:资源:操作`
4. **密码加密**：使用国密 SM3 算法加密存储
5. **授权层级**：`auth_level` 用于数据权限控制，数值越大权限越高
