# 数据库表结构修复说明

## 问题描述

登录时返回 **500 Internal Server Error**

## 问题原因

数据库表结构与后端代码不匹配：

| 项目 | 数据库 | 后端代码 | 状态 |
|------|--------|----------|------|
| 用户标识字段 | `username` | `user_code` | ❌ 不匹配 |
| 查询 SQL | - | `SELECT * FROM sys_user WHERE user_code = ?` | ❌ 字段不存在 |

**错误日志**:
```
com.mysql.jdbc.exceptions.jdbc4.MySQLSyntaxErrorException: 
Unknown column 'user_code' in 'where clause'
```

## 数据库现状

修复前的 `sys_user` 表结构：

```sql
+-------------+--------------+------+-----+-------------------+
| Field       | Type         | Null | Key | Default           |
+-------------+--------------+------+-----+-------------------+
| user_id     | varchar(32)  | NO   | PRI | NULL              |
| username    | varchar(50)  | NO   | UNI | NULL              | ← 只有这个
| password    | varchar(128) | NO   |     | NULL              |
| real_name   | varchar(50)  | NO   |     | NULL              |
| org_id      | varchar(32)  | NO   | MUL | NULL              |
| auth_level  | tinyint      | NO   |     | 1                 |
| user_status | tinyint      | NO   |     | 1                 |
| create_time | datetime     | NO   |     | CURRENT_TIMESTAMP |
| update_time | datetime     | NO   |     | CURRENT_TIMESTAMP |
+-------------+--------------+------+-----+-------------------+
```

## 修复方案

### 方案 1：添加 user_code 字段（✅ 已采用）

**优点**:
- 保留原有 username 字段
- 向后兼容
- 不影响现有业务

**执行 SQL**:

```sql
-- 1. 添加 user_code 字段
ALTER TABLE sys_user ADD COLUMN user_code VARCHAR(50) COMMENT '用户工号' AFTER user_id;

-- 2. 添加索引
ALTER TABLE sys_user ADD INDEX idx_user_code (user_code);

-- 3. 添加唯一约束
ALTER TABLE sys_user ADD UNIQUE KEY uk_user_code (user_code);

-- 4. 复制 username 到 user_code
UPDATE sys_user SET user_code = username WHERE user_code IS NULL;
```

### 方案 2：重命名 username 为 user_code

**优点**:
- 字段名统一
- 不需要额外字段

**缺点**:
- 可能影响其他依赖 username 字段的代码
- 需要修改 schema.sql

**执行 SQL**:

```sql
ALTER TABLE sys_user CHANGE COLUMN username user_code VARCHAR(50) NOT NULL;
```

## 常见问题

### Q1: 还是返回 500 错误

**原因**: 后端服务缓存或连接池问题

**解决**:
1. 重启后端服务
2. 检查后端日志
3. 确认数据库连接正常

### Q2: 登录成功但返回空数据

**原因**: 用户权限或角色配置问题

**解决**:
1. 检查 `sys_user_role` 表是否有对应角色
2. 检查 `sys_role_menu` 表是否有菜单权限
3. 查看后端日志确认查询结果
