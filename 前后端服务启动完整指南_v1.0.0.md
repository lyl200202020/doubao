## 服务启动

### 启动顺序

1. 启动数据库服务（MySQL、Redis）
2. 启动后端服务
3. 启动前端服务

### 后端服务启动

#### 方式一：使用JAR包启动（推荐）

```bash
# 进入启动模块目录
cd d:\study\doubao\customer-admin\customer-admin-platform\platform-start

# 启动服务（开发环境）
java -jar target/platform-start-1.0.0-SNAPSHOT.jar --spring.profiles.active=dev

# 启动服务（生产环境）
java -jar target/platform-start-1.0.0-SNAPSHOT.jar --spring.profiles.active=prod

# 指定JVM参数启动
java -Xms512m -Xmx1024m -jar target/platform-start-1.0.0-SNAPSHOT.jar --spring.profiles.active=dev
```

#### 方式二：使用Maven启动

```bash
# 进入启动模块目录
cd d:\study\doubao\customer-admin\customer-admin-platform\platform-start

# 启动服务
mvn clean spring-boot:run -Dspring-boot.run.profiles=dev

# 跳过测试启动
mvn clean spring-boot:run -Dspring-boot.run.profiles=dev -DskipTests
```

#### 方式三：在IDEA中启动

1. 打开IDEA，并导入项目
2. 找到 `PlatformApplication` 主类
3. 右键点击，选择 "Run 'PlatformApplication'"
4. 在运行配置中设置 `spring.profiles.active=dev`

### 前端服务启动

#### 方式一：使用npm启动（推荐）

```bash
# 进入前端项目目录
cd d:\study\doubao\customer-admin-web

# 启动开发服务器
npm run dev

# 或使用yarn
yarn dev

# 或使用pnpm
pnpm dev
```

#### 方式二：在VSCode中启动

1. 打开VSCode，并打开前端项目
2. 打开终端（Ctrl + `）
3. 运行 `npm run dev`

### 后台启动（Windows）

```bash
# 后端后台启动
start /B java -jar target/platform-start-1.0.0-SNAPSHOT.jar --spring.profiles.active=dev > backend.log 2>&1

# 前端后台启动
start /B npm run dev > frontend.log 2>&1
```

## 服务停止

### 停止后端服务

#### 方式一：使用Ctrl+C

在运行后端服务的终端窗口中按 `Ctrl + C`

#### 方式二：查找并终止进程

```bash
# 查找Java进程
tasklist | findstr java

# 终止指定进程（替换PID为实际进程ID）
taskkill /F /PID <PID>

# 终止所有Java进程
taskkill /F /IM java.exe
```

#### 方式三：使用端口查找进程

```bash
# 查找占用8080端口的进程
netstat -ano | findstr :8080

# 终止进程
taskkill /F /PID <PID>
```

### 停止前端服务

#### 方式一：使用Ctrl+C

在运行前端服务的终端窗口中按 `Ctrl + C`

#### 方式二：查找并终止进程

```bash
# 查找Node进程
tasklist | findstr node

# 终止指定进程
taskkill /F /PID <PID>

# 终止所有Node进程
taskkill /F /IM node.exe
```

## 常见问题

### 问题1：后端启动失败 - 端口被占用

**错误信息**：
```
Web server failed to start. Port 8080 was already in use.
```

**解决方案**：
```bash
# 查找占用端口的进程
netstat -ano | findstr :8080

# 终止进程
taskkill /F /PID <PID>

# 或修改配置文件中的端口
# 在 application.yml 中修改 server.port
```

### 问题2：数据库连接失败

**错误信息**：
```
Communications link failure
```

**解决方案**：
1. 检查MySQL服务是否启动
2. 检查数据库连接配置是否正确
3. 检查网络连接是否正常
4. 检查防火墙设置

### 问题3：Redis连接失败

**错误信息**：
```
Unable to connect to Redis
```

**解决方案**：
1. 检查Redis服务是否启动
2. 检查Redis连接配置是否正确
3. 测试Redis连接：
```bash
redis-cli -h 113.221.57.26 -p 6379 -a mm123456
```

### 问题4：前端无法访问后端API

**错误信息**：
```
ECONNREFUSED /api/xxx
```

**解决方案**：
1. 检查后端服务是否启动
2. 检查vite.config.ts中的代理配置
3. 确认代理目标端口正确（应为8080）
4. 检查浏览器控制台的网络请求

### 问题5：前端页面空白

**可能原因**：
- 路由配置错误
- 组件加载失败
- API请求失败

**解决方案**：
1. 检查浏览器控制台错误信息
2. 检查路由配置是否正确
3. 检查API请求是否成功
4. 清除浏览器缓存

### 问题6：登录失败

**可能原因**：
- 用户名或密码错误
- 数据库连接问题
- Redis连接问题
- JWT配置问题

**解决方案**：
1. 检查后端日志
2. 验证数据库用户数据
3. 检查Redis连接
4. 检查JWT配置

### 问题7：Maven依赖下载失败

**错误信息**：
```
Could not resolve dependencies
```

**解决方案**：
1. 配置Maven镜像源
2. 检查网络连接
3. 清理本地缓存：
```bash
mvn clean install -U
```

### 问题8：npm依赖安装失败

**错误信息**：
```
npm ERR! network timeout
```

**解决方案**：
1. 使用淘宝镜像：
```bash
npm config set registry https://registry.npmmirror.com
```
2. 清除npm缓存：
```bash
npm cache clean --force
```
3. 重新安装依赖
