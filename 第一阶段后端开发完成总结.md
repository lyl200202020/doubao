## 三、技术架构

### 3.1 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 语言 | JDK | 17 |
| 框架 | Spring Boot | 3.3.5 |
| 微服务 | Spring Cloud Alibaba | 2023.0.1.2 |
| RPC框架 | Dubbo | 3.2.14 |
| 注册/配置 | Nacos | 2.2.x |
| ORM | MyBatis-Plus | 3.5.7 |
| 数据库 | MySQL | 8.0.x |
| 缓存 | Redis | 7.0.x |
| 安全框架 | Spring Security | 6.x |
| 国密算法 | BouncyCastle | 1.77 |
| 文档 | SpringDoc OpenAPI | 2.5.0 |

### 3.2 架构模式

采用**DDD领域驱动设计 + 微服务架构**：

```
├── Domain（领域层）
│   ├── entity/        实体类
│   ├── repository/    仓储接口
│   └── service/       领域服务
├── Application（应用层）
│   ├── service/       应用服务
│   └── command/       命令对象
├── Infrastructure（基础设施层）
│   ├── dao/           DAO接口
│   └── repository/    仓储实现
└── Interface（接口层）
    ├── controller/    控制器
    ├── dto/           数据传输对象
    └── vo/            视图对象
```

## 六、API接口

### 6.1 认证接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 登录 | POST | /api/auth/login | 用户登录 |
| 退出 | POST | /api/auth/logout | 用户退出 |
| 当前用户 | GET | /api/auth/current | 获取当前登录用户信息 |

### 6.2 接口示例

**登录请求**
```json
POST /api/auth/login
Content-Type: application/json

{
    "username": "admin",
    "password": "123456"
}
```

**登录响应**
```json
{
    "code": 200,
    "msg": "操作成功",

        "username": "admin",
        "realName": "系统管理员",
        "orgId": "ORG001",
        "authLevel": 16
    },
    "timestamp": 1709625600000
}
```

## 八、启动说明

### 8.1 前置条件

1. MySQL 8.0+ 运行中
2. Redis 7.0+ 运行中
3. Nacos 2.2+ 运行中（可选）

### 8.2 启动步骤

1. **执行SQL脚本**
   ```bash
   mysql -u root -p < customer-admin/sql/schema.sql
   ```

2. **修改配置**（可选）
   - 编辑 `platform-start/src/main/resources/application-dev.yml`
   - 修改数据库连接信息
   - 修改Redis连接信息

3. **启动服务**
   - 运行 `PlatformApplication.java`
   - 服务端口：8080

4. **访问接口文档**
   - Swagger UI: http://localhost:8080/swagger-ui.html
   - OpenAPI: http://localhost:8080/v3/api-docs
