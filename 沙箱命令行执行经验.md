## MySQL 命令

### 踩坑记录

#### ❌ 错误1：PowerShell 不支持 `<` 重定向

```bash
# 错误写法
mysql -h 113.221.57.26 -u app -p123456 cust < schema.sql
```

**错误信息：**
```
"<" 运算符是为将来使用而保留的
```

**原因：** PowerShell 中 `<` 重定向不被支持

---

#### ❌ 错误2：命令行参数中带引号被错误解析

```bash
# 错误写法
mysql -h 113.221.57.26 -u app -p123456 -e "SELECT VERSION();"
mysql -h 113.221.57.26 -u app -p123456 -e "SHOW DATABASES"
```

**错误信息：**
```
unexpected argument 'VERSION();' found
unexpected argument 'DATABASES' found
```

**原因：** Windows 环境下通过沙箱执行命令时，引号内的参数被错误解析

---

#### ❌ 错误3：MySQL 客户端命令参数不带 `-e` 时需要用管道

```bash
# 错误写法
mysql -h 113.221.57.26 -u app -p123456 cust SHOW TABLES
```

**原因：** MySQL 客户端需要通过 stdin 接收 SQL 语句

---

#### ❌ 错误4：SQL文件中含中文注释导致编码错误

```sql
-- 含中文注释的SQL
CREATE TABLE sys_user (
    ...
    user_status TINYINT NOT NULL DEFAULT 1 COMMENT '状态:0-禁用,1-启用'
);
```

**错误信息：**
```
ERROR 1064: You have an error in your SQL syntax... near '????0-???'
```

**原因：** Windows 环境下通过 `type` 管道传输中文时编码问题

**解决方案：** 移除 SQL 文件中的中文注释

---

### ✅ 正确写法

#### 方式1：使用管道（推荐）

```bash
# 适用于执行 SQL 文件
type d:\path\to\schema.sql | mysql -h 113.221.57.26 -u app -p123456 cust

# 适用于执行单条SQL
echo SELECT VERSION() | mysql -h 113.221.57.26 -u app -p123456
```

#### 方式2：创建临时SQL文件

```sql
-- verify_data.sql
SELECT username, real_name, auth_level FROM sys_user;
```

```bash
type d:\study\doubao\sql\verify_data.sql | mysql -h 113.221.57.26 -u app -p123456 cust
```

#### 方式3：MySQL交互式客户端（最推荐）

```bash
# 进入MySQL客户端
mysql -h 113.221.57.26 -u app -p123456 cust

# 在客户端内执行
mysql> source d:/study/doubao/customer-admin/sql/schema.sql
mysql> SELECT * FROM sys_user;
```

## Maven 命令

### 踩坑记录

#### ❌ 错误1：PowerShell 不支持 `&&` 命令连接符

```powershell
# 错误写法
cd d:\study\doubao\customer-admin\customer-admin-platform\platform-biz && mvn compile -DskipTests
```

**错误信息：**
```
所在位置 行:1 字符:72
+ ... doubao\customer-admin\customer-admin-platform\platform-biz && mvn com ...
+                                                                ~~
标记"&&"不是此版本中的有效语句分隔符。
```

**原因：** PowerShell 中 `&&` 不是有效的语句分隔符，这是 Bash/CMD 的语法

**解决方案：** 使用分号 `;` 代替 `&&`

```powershell
# 正确写法
Set-Location d:\study\doubao\customer-admin\customer-admin-platform\platform-biz; mvn compile -DskipTests

# 或者简写
cd d:\study\doubao\customer-admin\customer-admin-platform\platform-biz; mvn compile -DskipTests
```

---

#### ❌ 错误2：Maven 命令未找到

```bash
# 错误写法
mvn compile -DskipTests
```

**错误信息：**
```
无法将"mvn"项识别为 cmdlet、函数、脚本文件或可运行程序的名称
```

**原因：** Maven 未配置到系统环境变量

**解决方案：** 
1. 配置 Maven 到系统 PATH
2. 或使用 Maven Wrapper (mvnw)

---

### ✅ 正确写法

#### 方式1：使用分号分隔命令

```powershell
# 切换目录并编译
Set-Location d:\study\doubao\customer-admin\customer-admin-parent; mvn clean install -DskipTests

# 从指定模块继续编译（-rf 参数）
Set-Location d:\study\doubao\customer-admin\customer-admin-parent; mvn clean install -DskipTests -rf :platform-start
```

#### 方式2：直接指定项目路径

```powershell
# 使用 -f 参数指定 pom.xml 位置
mvn -f d:\study\doubao\customer-admin\customer-admin-parent\pom.xml clean install -DskipTests
```

#### 方式3：PowerShell 7+ 支持 &&

```powershell
# 如果使用 PowerShell 7 或更高版本，支持 && 语法
cd d:\study\doubao\customer-admin\customer-admin-parent && mvn clean install -DskipTests
```

> **注意：** Windows 默认的 PowerShell 5.1 不支持 `&&`，需要升级到 PowerShell 7+ 才能使用

## HTTP 请求命令

### 踩坑记录

#### ❌ 错误1：URL 中包含 `&` 字符被解析为后台任务

```powershell
# 错误写法（双引号包含&）
Invoke-WebRequest -Uri "http://113.221.57.26:8848/nacos/v1/cs/configs?dataId=app.yml&group=DEFAULT_GROUP"
```

**错误信息：**
```
不允许使用与号(&)。& 运算符是为将来使用而保留的
```

**原因：** PowerShell 中 `&` 是后台任务运算符

**解决方案：**
1. 将 URL 用单引号包裹
2. 或使用 URL 编码 `%26` 替代 `&`

---

### ✅ 正确写法

#### 方式1：使用单引号（推荐）

```powershell
# 简单 GET 请求
Invoke-WebRequest -Uri "http://113.221.57.26:8848/nacos/v1/console/health/readiness" -Method GET -UseBasicParsing | Select-Object StatusCode

# 带查询参数（使用单引号避免&解析问题）
Invoke-WebRequest -Uri 'http://113.221.57.26:8848/nacos/v1/cs/configs?dataId=application.yml&group=DEFAULT_GROUP' -Method GET -UseBasicParsing | Select-Object StatusCode
```

#### 方式2：URL 编码替代 &

```powershell
# 使用 %26 替代 &
Invoke-WebRequest -Uri "http://113.221.57.26:8848/nacos/v1/cs/configs?dataId=app.yml%26group=DEFAULT_GROUP" -Method GET -UseBasicParsing
```

#### 方式3：健康检查常用命令

```powershell
# Nacos 健康检查
Invoke-WebRequest -Uri "http://113.221.57.26:8848/nacos/v1/console/health/readiness" -Method GET -UseBasicParsing | Select-Object StatusCode

# 获取配置
Invoke-WebRequest -Uri 'http://113.221.57.26:8848/nacos/v1/cs/configs?dataId=application.yml&group=DEFAULT_GROUP' -Method GET -UseBasicParsing
```

|------|------|

## SQL文件编码规范

在 Windows 环境下通过 `type` 管道执行 SQL 时：

1. **避免使用中文注释**，或使用英文
2. **避免使用特殊字符**
3. **推荐格式：**

```sql
USE cust;

CREATE TABLE IF NOT EXISTS sys_user (
    user_id VARCHAR(32) PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(128) NOT NULL,
    real_name VARCHAR(50) NOT NULL,
    org_id VARCHAR(32) NOT NULL,
    auth_level TINYINT NOT NULL DEFAULT 1,
    user_status TINYINT NOT NULL DEFAULT 1,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_org (org_id),
    INDEX idx_username (username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

INSERT INTO sys_user (user_id, username, password, real_name, org_id, auth_level, user_status) VALUES
('USER001', 'admin', 'e10adc3949ba59abbe56e057f20f883e', 'AdminUser', 'ORG001', 16, 1);
```
