// AuthController.java
@PostMapping("/logout")
public Result<Void> logout(HttpServletRequest request) {
    String userId = (String) request.getAttribute(SecurityConstants.CURRENT_USER_ID);
    String token = request.getHeader(SecurityConstants.TOKEN_HEADER);
    if (token != null && token.startsWith(SecurityConstants.TOKEN_PREFIX)) {
        token = token.substring(SecurityConstants.TOKEN_PREFIX.length());
        if (userId != null) {
            authApplicationService.logout(userId, token);
        }
    }
    return Result.success();
}

// AuthApplicationService.java
public void logout(String userId, String token) {
    stringRedisTemplate.delete(SecurityConstants.CURRENT_USER + ":" + userId);
    stringRedisTemplate.opsForValue().set(
        SecurityConstants.TOKEN_BLACKLIST_PREFIX + token, "1",
        SecurityConstants.TOKEN_EXPIRE_TIME, TimeUnit.MILLISECONDS);
}

#### BUG-009：SM3密码验证存在时序攻击风险

| 项目 | 内容 |
|------|------|
| 缺陷编号 | BUG-009 |
| 缺陷等级 | P2-一般 |
| 修复状态 | ✅ 已修复 |
| 验证结果 | 通过 |

**问题描述**：
AuthDomainService.java 密码比对使用 `equals()` 方法，存在时序攻击风险。

**修复方案**：
使用常量时间比对方法。

**修复代码位置**：
- [SM3Util.java:69-80](file:///d:/study/doubao/customer-admin/customer-admin-common/common-core/src/main/java/com/company/admin/common/util/SM3Util.java#L69-L80)

**修复后代码**：
```java
public static boolean verify(String data, String hashValue) {
    if (data == null || hashValue == null) {
        return false;
    }
    String dataHash = hash(data);
    return dataHash != null && constantTimeEquals(dataHash, hashValue);
}

private static boolean constantTimeEquals(String a, String b) {
    if (a == null || b == null || a.length() != b.length()) {
        return false;
    }
    byte[] aBytes = a.getBytes(StandardCharsets.UTF_8);
    byte[] bBytes = b.getBytes(StandardCharsets.UTF_8);
    int result = 0;
    for (int i = 0; i < aBytes.length; i++) {
        result |= aBytes[i] ^ bBytes[i];
    }
    return result == 0;
}
```

**验证方法**：代码审查，确认使用常量时间比对。
