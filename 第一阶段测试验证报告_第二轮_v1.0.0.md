## 三、缺陷清单

### 3.1 P0级缺陷（阻塞发布）

#### BUG-011：登录响应orgName和permissions字段为空

| 项目 | 内容 |
|------|------|
| 缺陷编号 | BUG-011 |
| 缺陷等级 | **P0-阻塞** |
| 缺陷模块 | 认证模块 |
| 发现时间 | 2026-03-07 |
| 影响范围 | 登录功能、权限控制 |

**问题描述**：
登录接口返回的 `orgName` 和 `permissions` 字段为空，但数据库中存在对应数据。

**验证数据**：

| 字段 | 预期值 | 实际值 | 数据库验证 |
|------|--------|--------|------------|
| orgName | 总行 | 空 | ✅ 存在 |
| permissions | 非空数组 | 空/null | ✅ 存在84条角色菜单关联 |

**问题定位**：
- [AuthApplicationService.java](file:///d:/study/doubao/customer-admin/customer-admin-platform/platform-biz/src/main/java/com/company/admin/platform/sys/application/service/AuthApplicationService.java) 第65-70行
- 查询逻辑可能存在问题，或数据未正确映射

**影响**：
- 前端无法显示用户机构名称
- 前端无法获取用户权限列表
- 权限控制功能无法实现

**修复建议**：
1. 检查 SysOrgDao.findByOrgId() 查询是否正确执行
2. 检查 SysMenuDao.findPermissionsByUserId() 查询是否正确执行
3. 添加日志输出确认查询结果

---

#### BUG-012：获取当前用户接口返回500错误

| 项目 | 内容 |
|------|------|
| 缺陷编号 | BUG-012 |
| 缺陷等级 | **P0-阻塞** |
| 缺陷模块 | 用户模块 |
| 发现时间 | 2026-03-07 |
| 影响范围 | 前端首页显示 |

**问题描述**：
GET /api/auth/current 接口返回HTTP 500错误，后端日志显示NullPointerException。

**错误堆栈**：
```
java.lang.NullPointerException: null
    at com.company.admin.platform.sys.application.service.UserApplicationService.getCurrentUser(UserApplicationService.java:xx)
```

**问题定位**：
- [UserApplicationService.java](file:///d:/study/doubao/customer-admin/customer-admin-platform/platform-biz/src/main/java/com/company/admin/platform/sys/application/service/UserApplicationService.java)

**可能原因**：
1. sysUserDao.selectById() 返回null
2. 某些字段为null导致后续操作异常
3. MyBatis映射配置问题

**影响**：
- 前端无法获取当前用户信息
- 首页无法正常显示用户数据

**修复建议**：
1. 添加空值检查
2. 检查MyBatis映射配置
3. 添加详细日志定位问题

### 3.2 P1级缺陷（影响功能）

#### BUG-013：获取菜单树接口返回认证失败

| 项目 | 内容 |
|------|------|
| 缺陷编号 | BUG-013 |
| 缺陷等级 | **P1-严重** |
| 缺陷模块 | 菜单模块 |
| 发现时间 | 2026-03-07 |
| 影响范围 | 前端菜单显示 |

**问题描述**：
GET /api/sys/menu/tree 接口返回认证失败（code: 5001），但Token有效且已设置SecurityContext。

**错误响应**：
```json
{
    "code": 5001,
    "msg": "认证失败",
    "success": false
}
```

**问题定位**：
- [JwtAuthenticationFilter.java](file:///d:/study/doubao/customer-admin/customer-admin-common/common-security/src/main/java/com/company/admin/security/filter/JwtAuthenticationFilter.java)
- [SecurityConfig.java](file:///d:/study/doubao/customer-admin/customer-admin-common/common-security/src/main/java/com/company/admin/security/config/SecurityConfig.java)

**可能原因**：
1. SecurityContext未正确传递到后续Filter
2. 认证异常处理逻辑问题
3. 接口路径未正确配置白名单

**影响**：
- 前端无法获取菜单树
- 侧边栏菜单无法显示

**修复建议**：
1. 检查SecurityConfig中接口路径配置
2. 确认SecurityContext在Filter链中正确传递
3. 添加日志确认认证流程
