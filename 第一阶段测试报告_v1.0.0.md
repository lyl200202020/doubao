### 2.1 P0级缺陷（阻塞发布）

#### BUG-001：JWT Token字段名不一致导致认证失败

| 项目 | 内容 |
|------|------|
| 缺陷编号 | BUG-001 |
| 缺陷等级 | **P0-阻塞** |
| 缺陷模块 | 认证模块 |
| 发现时间 | 2026-03-07 |
| 影响范围 | 所有需要Token认证的接口 |

**问题描述**：

[JwtUtil.java](file:///d:/study/doubao/customer-admin/customer-admin-common/common-security/src/main/java/com/company/admin/security/jwt/JwtUtil.java) 生成Token时使用的Claim字段名与 [JwtAuthenticationFilter.java](file:///d:/study/doubao/customer-admin/customer-admin-common/common-security/src/main/java/com/company/admin/security/filter/JwtAuthenticationFilter.java) 解析Token时使用的字段名不一致。

**代码对比**：

| 文件 | 字段定义 | 实际值 |
|------|----------|--------|
| JwtUtil.java | `CLAIM_USER_ID = "userId"` | userId |
| SecurityConstants.java | `CURRENT_USER_ID = "CURRENT_USER_ID"` | CURRENT_USER_ID |

**问题代码位置**：

```java
// JwtUtil.java 第47行 - 生成Token时使用
.claim(CLAIM_USER_ID, userId)  // 字段名: "userId"

// JwtAuthenticationFilter.java 第47行 - 解析Token时使用
String userId = claims.get(SecurityConstants.CURRENT_USER_ID, String.class);  // 字段名: "CURRENT_USER_ID"
```

**影响**：
- Token生成后无法正确解析
- 所有需要认证的接口都会返回null
- 用户登录后无法访问受保护资源

**修复建议**：

修改 [JwtAuthenticationFilter.java](file:///d:/study/doubao/customer-admin/customer-admin-common/common-security/src/main/java/com/company/admin/security/filter/JwtAuthenticationFilter.java) 第47-51行：

```java
// 修改前
String userId = claims.get(SecurityConstants.CURRENT_USER_ID, String.class);
String username = claims.get(SecurityConstants.CURRENT_USERNAME, String.class);
String realName = claims.get(SecurityConstants.CURRENT_REAL_NAME, String.class);
String orgId = claims.get(SecurityConstants.CURRENT_ORG_ID, String.class);
Integer authLevel = claims.get(SecurityConstants.CURRENT_AUTH_LEVEL, Integer.class);

// 修改后
String userId = claims.get(JwtUtil.CLAIM_USER_ID, String.class);
String username = claims.get(JwtUtil.CLAIM_USERNAME, String.class);
String realName = claims.get(JwtUtil.CLAIM_REAL_NAME, String.class);
String orgId = claims.get(JwtUtil.CLAIM_ORG_ID, String.class);
Integer authLevel = claims.get(JwtUtil.CLAIM_AUTH_LEVEL, Integer.class);
```

#### BUG-006：退出登录未清除Redis Token

| 项目 | 内容 |
|------|------|
| 缺陷编号 | BUG-006 |
| 缺陷等级 | **P1-严重** |
| 缺陷模块 | 认证模块 |
| 发现时间 | 2026-03-07 |

**问题描述**：

[AuthController.java](file:///d:/study/doubao/customer-admin/customer-admin-platform/platform-biz/src/main/java/com/company/admin/platform/sys/api/controller/AuthController.java) 第32-38行，退出登录仅打印日志，未调用业务层清除Token。

**问题代码**：

```java
@PostMapping("/logout")
public Result<Void> logout(HttpServletRequest request) {
    String token = request.getHeader(SecurityConstants.TOKEN_HEADER);
    if (token != null && token.startsWith(SecurityConstants.TOKEN_PREFIX)) {
        token = token.substring(SecurityConstants.TOKEN_PREFIX.length());
        log.info("User logout, token: {}", token.substring(0, Math.min(20, token.length())) + "...");
    }
    return Result.success();  // ❌ 未清除Redis中的Token
}
```

**影响**：
- Token在Redis中仍然有效
- 用户退出后Token仍可使用
- 安全风险

**修复建议**：

调用 AuthApplicationService.logout() 清除Token：

```java
@PostMapping("/logout")
public Result<Void> logout(HttpServletRequest request) {
    String token = request.getHeader(SecurityConstants.TOKEN_HEADER);
    if (token != null && token.startsWith(SecurityConstants.TOKEN_PREFIX)) {
        token = token.substring(SecurityConstants.TOKEN_PREFIX.length());
        Claims claims = JwtUtil.parseToken(token);
        String userId = claims.get(JwtUtil.CLAIM_USER_ID, String.class);
        authApplicationService.logout(userId);
    }
    return Result.success();
}
```

#### BUG-009：SM3密码验证存在时序攻击风险

| 项目 | 内容 |
|------|------|
| 缺陷编号 | BUG-009 |
| 缺陷等级 | **P2-一般** |
| 缺陷模块 | 安全模块 |
| 发现时间 | 2026-03-07 |

**问题描述**：

[AuthDomainService.java](file:///d:/study/doubao/customer-admin/customer-admin-platform/platform-biz/src/main/java/com/company/admin/platform/sys/domain/service/AuthDomainService.java) 第31行，密码比对使用 `equals()` 方法。

**问题代码**：

```java
if (!hashedPassword.equals(user.getPassword())) {
    throw new BusinessException(ErrorCode.USER_PASSWORD_ERROR);
}
```

**影响**：
- 存在时序攻击风险
- 攻击者可通过响应时间推断密码

**修复建议**：

使用常量时间比对：

```java
import java.security.MessageDigest;

// 使用MessageDigest.isEqual进行常量时间比对
if (!MessageDigest.isEqual(hashedPassword.getBytes(), user.getPassword().getBytes())) {
    throw new BusinessException(ErrorCode.USER_PASSWORD_ERROR);
}
```
