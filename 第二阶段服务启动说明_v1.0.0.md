## 修复内容

### 问题描述
前端无法访问后端，代理错误 `ECONNREFUSED`

### 问题原因
前端 Vite 配置的代理目标端口为 `8081`，但后端实际运行在 `8080` 端口

### 解决方案
修改 `vite.config.ts` 中的代理配置：

```typescript
// 修改前
proxy: {
  '/api': {
    target: 'http://localhost:8081',  // ❌ 错误端口
    changeOrigin: true
  }
}

// 修改后
proxy: {
  '/api': {
    target: 'http://localhost:8080',  // ✅ 正确端口
    changeOrigin: true
  }
}
```

## 常见问题

### Q1: 前端访问后端 404

**原因**: 后端服务未启动或端口配置错误

**解决**: 
1. 确认后端服务已启动
2. 检查 `vite.config.ts` 中的代理配置
3. 检查后端 `application-dev.yml` 中的端口配置

### Q2: 前端页面空白

**原因**: 可能是路由配置或组件加载问题

**解决**:
1. 检查浏览器控制台错误信息
2. 确认路由配置正确
3. 确认组件文件存在

### Q3: 登录失败

**原因**: 可能是数据库连接问题或认证配置问题

**解决**:
1. 检查后端日志
2. 确认数据库连接正常
3. 确认 Redis 连接正常
4. 检查 JWT 配置
