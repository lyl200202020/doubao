## 5 详细功能说明

### 5.1 登录功能

#### 5.1.1 功能描述

| 项目 | 说明 |
|------|------|
| 功能名称 | 用户登录 |
| 入口 | 登录页 |
| 触发方式 | 输入用户名密码，点击登录按钮 |

#### 5.1.2 业务流程

```
1. 用户输入用户名、密码
2. 前端调用登录API
3. 后端验证用户名密码（SM3加密比对）
4. 验证成功生成JWT Token
5. 前端存储Token，跳转首页
6. 验证失败显示错误信息
```

#### 5.1.3 接口设计

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 登录 | POST | /api/auth/login | 用户名密码登录 |

#### 5.1.4 请求参数

```json
{
  "username": "admin",
  "password": "admin123"
}
```

#### 5.1.5 返回结果

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "userInfo": {
      "userId": "1001",
      "username": "admin",
      "realName": "系统管理员",
      "orgId": "0000",
      "orgName": "总行",
      "authLevel": 16
    }
  }
}
```

### 5.2 退出登录功能

#### 5.2.1 功能描述

| 项目 | 说明 |
|------|------|
| 功能名称 | 退出登录 |
| 入口 | 顶部导航栏 |
| 触发方式 | 点击退出按钮 |

#### 5.2.2 业务流程

```
1. 用户点击退出按钮
2. 前端调用退出API
3. 后端清除Token（可选）
4. 前端清除Token和用户信息
5. 跳转登录页
```

#### 5.2.3 接口设计

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 退出 | POST | /api/auth/logout | 退出登录 |
