## 二、遇到的问题及修复

### 1. 登录500错误

**问题描述**：用户登录时出现500服务器错误

**原因分析**：
- Redis连接配置路径错误
- 配置文件中使用 `spring.redis`，但Spring Boot 3.x应该使用 `spring.data.redis`

**修复方案**：
- 修改文件：`customer-admin-platform/platform-start/src/main/resources/application-dev.yml`
- 将 `spring.redis` 改为 `spring.data.redis`

**修复代码**：
```yaml
spring
  data:
    redis:
      host: 113.221.57.26
      port: 6379
      password: mm123456
      database: 0
      timeout: 5000ms
```

### 2. 用户管理空白页

**问题描述**：登录后点击"用户管理"菜单，页面显示空白

**原因分析**：
- 路由组件路径错误
- 使用了绝对路径 `/src/views${menu.menuUrl}/index.vue`，应该使用别名路径 `@/views${menu.menuUrl}/index.vue`

**修复方案**：
- 修改文件：`customer-admin-web/src/utils/router.ts`
- 将组件路径从 `/src/views` 改为 `@/views`

**修复代码**：
```typescript
const componentPath = `@/views${menu.menuUrl}/index.vue`
route.component = modules[componentPath] || (() => import('@/views/error/404.vue'))
```

### 3. UserForm响应式代理错误

**问题描述**：UserForm.vue中出现响应式代理错误

**原因分析**：
- 表单字段名称与后端DTO不匹配
- 前端使用 `username` 和 `realName`，后端期望 `userCode` 和 `userName`

**修复方案**：
- 修改文件：`customer-admin-web/src/views/sys/user/components/UserForm.vue`
- 更新表单字段名称、验证规则和重置函数

**修复代码**：
```vue
<el-form-item label="用户工号" prop="userCode">
  <el-input v-model="formData.userCode" placeholder="请输入用户工号" :disabled="isEdit" />
</el-form-item>
<el-form-item label="用户姓名" prop="userName">
  <el-input v-model="formData.userName" placeholder="请输入用户姓名" />
</el-form-item>
```

### 4. UserForm初始化顺序错误

**问题描述**：UserForm.vue中出现"Cannot access 'resetForm' before initialization"错误

**原因分析**：
- JavaScript暂时性死区错误
- `resetForm`函数在被定义之前就被调用
- `watch`钩子中调用了`resetForm`，但函数定义在watch之后

**修复方案**：
- 修改文件：`customer-admin-web/src/views/sys/user/components/UserForm.vue`
- 将`resetForm`函数定义移到`watch`之前

**修复代码**：
```typescript
const resetForm = () => {
  formData.value = {
    userId: undefined,
    userCode: '',
    userName: '',
    password: '',
    orgId: '',
    authLevel: 1,
    userStatus: 1
  }
}

watch(() => props.formData, (newVal) => {
  if (newVal) {
    formData.value = { ...newVal }
  } else {
    resetForm()
  }
}, { immediate: true, deep: true })
```

### 5. 布局空白区域问题

**问题描述**：页面有大量空白区域，内容没有撑满屏幕

**原因分析**：
- `.app-main`区域有20px的内边距
- 布局没有使用flex布局充分利用屏幕空间

**修复方案**：
- 修改文件：`customer-admin-web/src/components/Layout/AppLayout.vue`
- 移除内边距，添加flex布局和最小高度

**修复代码**：
```scss
.app-main {
  flex: 1;
  overflow-y: auto;
  padding: 0;
  background-color: #f5f7fa;
  min-height: calc(100vh - 60px);
}
```

### 6. 权限按钮显示问题

**问题描述**：用户管理页面的操作按钮不显示

**原因分析**：
- 权限指令正常工作
- 数据库中存在所有需要的权限
- 权限数据加载正常
- 问题可能是浏览器缓存或权限数据加载时机

**检查结果**：
- 数据库查询显示admin用户拥有以下权限：
  - `sys:user:list` - 用户列表查看
  - `sys:user:add` - 新增用户
  - `sys:user:edit` - 编辑用户
  - `sys:user:delete` - 删除用户
  - `sys:user:resetPwd` - 重置密码
  - `sys:user:view` - 查看用户详情

**解决方案**：
- 强制刷新浏览器（Ctrl + F5）
- 重新登录确保权限数据正确加载

### 7. 表格右侧留白问题

**问题描述**：用户管理页面表格右侧有大量留白

**原因分析**：
- 表格列宽设置过小
- 没有充分利用表格宽度

**修复方案**：
- 修改文件：`customer-admin-web/src/views/sys/user/index.vue`
- 调整表格列宽，使用min-width让某些列自适应

**修复代码**：
```vue
<el-table-column prop="userCode" label="用户工号" width="150" />
<el-table-column prop="userName" label="用户名" width="150" />
<el-table-column prop="orgName" label="所属机构" min-width="200" />
<el-table-column prop="authLevel" label="授权级别" width="120" />
<el-table-column prop="userStatus" label="状态" width="100" />
<el-table-column label="操作" fixed="right" width="300" />
```

### 8. 按钮消失问题（多次出现）

**问题描述**：修复其他问题后，操作按钮又消失了

**原因分析**：
- 在优化布局时添加了 `overflow: hidden` 样式
- 导致按钮被裁剪或隐藏在可视区域之外

**修复方案**：
- 修改文件：`customer-admin-web/src/views/sys/user/index.vue`
- 移除 `.el-card` 和 `.el-card__body` 的 `overflow: hidden`

**修复代码**：
```scss
:deep(.el-card) {
  flex: 1;
  display: flex;
  flex-direction: column;
  height: 100%;
}

:deep(.el-card__body) {
  flex: 1;
  display: flex;
  flex-direction: column;
  padding: 20px;
}
```

4. **JavaScript**：注意函数定义和调用的顺序
5. **布局优化**：使用flex布局可以更好地利用屏幕空间
6. **权限控制**：权限数据加载要正确，避免按钮不显示
