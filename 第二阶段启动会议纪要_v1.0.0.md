《需求设计文档_客户运营管理后台系统_v1.0.0.md》第 4 章 - 功能模块设计

### 2.2 开发范围

| 版本 | 阶段 | 交付内容 |

### 2.3 功能清单

#### 机构管理（v0.0.2）

| 功能 | 说明 | 业务规则 |
|------|------|----------|
| 机构列表 | 树形展示所有机构 | 机构级别固定 4 级（0-总行、1-分行、2-支行、3-分理处） |
| 新增机构 | 选择上级机构，新增下级机构 | 新增机构级别 = 上级机构级别 + 1 |
| 修改机构 | 编辑机构信息 | 机构代码唯一，不可修改 |
| 删除机构 | 逻辑删除机构 | 删除前检查是否有下级机构或关联用户 |
| 启用/停用 | 启用或停用机构 | 停用机构级联停用下级机构 |

#### 用户管理（v0.0.3）

| 功能 | 说明 | 业务规则 |
|------|------|----------|
| 用户列表 | 分页展示用户，支持按机构、角色、状态筛选 | 用户工号唯一 |
| 新增用户 | 填写用户基本信息，绑定机构和角色 | 用户只能属于一个机构、一个角色 |
| 修改用户 | 编辑用户信息，可修改机构和角色 | 密码使用国密 SM3 加密 |
| 删除用户 | 逻辑删除用户 | 删除前检查是否有待办任务 |
| 重置密码 | 重置用户密码为默认密码 | 首次登录强制修改密码 |
| 启用/停用 | 启用或停用用户账号 | 停用后无法登录 |

#### 菜单管理（v0.0.3）

| 功能 | 说明 | 业务规则 |
|------|------|----------|
| 菜单树 | 树形展示所有菜单，支持拖拽排序 | 菜单类型：目录（1）、菜单（2）、按钮（3） |
| 新增菜单 | 选择父节点，新增目录/菜单/按钮 | 菜单层级最多 4 级 |
| 修改菜单 | 编辑菜单信息 | 按钮级别菜单需配置权限标识 |
| 删除菜单 | 删除菜单，级联删除子菜单 | 删除前检查是否被角色使用 |
| 分配角色 | 为菜单分配可访问的角色 | 一个菜单可绑定多个角色 |

#### 角色管理（v0.0.4）

| 功能 | 说明 | 业务规则 |
|------|------|----------|
| 角色列表 | 分页展示角色，支持按状态筛选 | 角色代码唯一 |
| 新增角色 | 填写角色基本信息 | 角色只控制菜单可见性，不参与授权审批 |
| 修改角色 | 编辑角色信息 | 删除前需解绑用户 |
| 删除角色 | 逻辑删除角色 | - |
| 分配菜单 | 为角色分配可见的菜单（多选） | 菜单包括目录、菜单、按钮 |

## 三、技术方案

### 3.1 技术架构

**后端**：
- 框架：SpringBoot 3.3 + MyBatis-Plus + Dubbo 3.2+
- 架构：DDD 四层架构（domain、application、infrastructure、api）
- 数据库：MySQL 8.0+
- 缓存：Redis 6.0+
- 安全：国密 SM3 密码加密

**前端**：
- 框架：Vue3 + Vite + TypeScript + Pinia
- UI 库：Element Plus
- 路由：Vue Router 4
- 构建：Vite 5
