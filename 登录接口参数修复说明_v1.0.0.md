# 登录接口参数修复说明

## 问题描述

前端登录请求返回 **400 Bad Request** 错误

## 问题原因

前后端登录接口参数名称不一致：

| 端 | 参数名 | 说明 |
|----|--------|------|
| **前端（修复前）** | `username` | ❌ 错误 |
| **后端** | `userCode` | ✅ 正确（用户工号） |
| **前端（修复后）** | `userCode` | ✅ 已修正 |

## 后端定义

**文件**: `AuthController.java`

```java
@PostMapping("/login")
public Result<LoginVO> login(@Valid @RequestBody LoginCommand command) {
    LoginVO loginVO = authApplicationService.login(command);
    return Result.success(loginVO);
}
```

**LoginCommand.java**:

```java
@Data
@Schema(description = "登录请求")
public class LoginCommand {
    
    @NotBlank(message = "用户工号不能为空")
    @Schema(description = "用户工号")
    private String userCode;  // ← 使用 userCode
    
    @NotBlank(message = "密码不能为空")
    @Schema(description = "密码")
    private String password;
}
```

## 常见问题

### Q1: 还是返回 400 怎么办？

**原因**: 浏览器缓存了旧代码

**解决**: 
1. 强制刷新页面（Ctrl + Shift + R 或 Ctrl + F5）
2. 清除浏览器缓存
3. 使用无痕模式打开

### Q2: 返回 401 Unauthorized

**原因**: 用户工号或密码错误

**解决**:
1. 确认用户工号正确（默认：`admin`）
2. 确认密码正确（默认：`123456`）
3. 检查后端日志查看详细错误

### Q3: 返回 500 Internal Server Error

**原因**: 后端服务异常

**解决**:
1. 检查后端日志
2. 确认数据库连接正常
3. 确认 Redis 连接正常
4. 重启后端服务
